# Finance-Stock-Video · 一个公司/主题 → 整套财经视频物料

把「一个 A 股公司（或一个财经问题）」变成《A股上市公司速览》系列的一整套可发布物料：官方数据核实 → 客观脚本 → 灵依配音 → 横版 + 竖版成片 → 4 比例封面 → 平台文案。目标是 **给个公司代码或主题就能一路产出**。

本 skill 是 **财经领域层**，坐在 **`blockframe-video`**（通用视频编排：合成 / 渲染 / 全比例封面 / 清单）之上，只补它没有的、财经独有的东西。视频怎么渲、封面怎么配全，交给 blockframe-video；本 skill 管**数据从哪来、口径怎么守、内容怎么排、系列长什么样**。

**项目目录结构**：一个选题一个项目目录（`studio/videos/<YYYYMMDD-slug>/`），横竖两版是里面的 `build-h` / `build-v` 两个子目录、共用上层 `audio/` `assets/` `fonts/`；`research.md` / `script.md` / `narration.txt` / `renders/` / `covers/` / `publish.md` / `blog.md` 各就各位。**别再散成 `-cn`/`-3x4`/`-16x9` 兄弟目录**。完整结构见 blockframe-video 的「项目目录结构」。

## 三条铁律（违反任何一条这期就是废的）

1. **数字必须核对官方原件，不信转述。** 下定期报告 PDF → `pdftotext -layout` → `grep` 核对每个关键数字。子 agent / 媒体给的数字一律回原件核。
2. **客观陈列，不构成投资建议。** 无估值、无目标价、无买卖暗示、无「值得关注 / 低估」这类倾向词。结尾固定免责定格。
3. **市场叙事必打标签。** 出口管制、国产替代、宏观利好这类**不是公司自己说的**归因，一律显式标注「市场分析 · 非公司口径」，并在画面上用醒目横幅隔离。

## 两种选题模板（按输入选）

**① 个股速览（输入 = 一个公司）** — 客观陈列，固定 8 分块：
开场卡 → 公司简介 → 主营业务 → 产品线（年报谱系 + 官网在售双源）→ 客户与场景 → 财务趋势（多季度）→ 前十大流通股东 → 收尾 + 免责。

## Step 1 · 数据采集与核实（本 skill 的核心，别省）

### 数据源优先级
1. **巨潮资讯 cninfo**（`cninfo.com.cn` / `dataclouds.cninfo.com.cn`）—— 定期报告 / 公告**原件 PDF**，最高优先。
2. **交易所披露**（上交所 / 深交所）、公司公告、公司官网（产品 / 品牌资料）。
3. **权威财经媒体**（财联社、证券时报、第一财经、e公司）—— 只作补充与时间线，标【第三方】。
4. **自媒体 / 研报观点** —— 谨慎，一律标【第三方】或【存疑】，绝不当事实陈述。

### 必挖的官方数据 checklist（个股速览 + 归因都从这里取）
- 营收 / 归母净利 / 扣非 / 同比、及扭亏/首盈时间点
- 综合毛利率；**分产品 / 分业务收入表**（含各行金额、同比、毛利率）
- **主要产品产量 / 销量 / 库存量表**（销量放量是「卖了多少」的官方硬数据）
- 分地区 / 分销售模式（直销 vs 经销）
- **前五大客户**：合计占比 + **逐个金额**（即使匿名）+ 新增/老客户说明
- **存货明细**（原材料 / 在产品 / 库存商品 / 发出商品，期末 vs 期初）+ 占总资产比 + 跌价
- 应收账款、合同负债、经营现金流；前五大 / 第一大**供应商**占采购比
- 研发投入及占比；定增 / 重大合同（「重大销售合同」勾选项，"不适用"=无单一大单）
- 风险因素章节（客户集中 / 供应链 / 存货 / 实体清单 / 竞争）照抄
- **前十大流通股东** + 股东户数 + 实控人持股（个股速览必备）

### 核实铁律（照做，别只信子 agent）
```bash
# 1) 从 cninfo 下年报/季报原件
curl -sSL -A "Mozilla/5.0" -o report.pdf "<cninfo PDF url>"
# 2) 抽文本
pdftotext -layout report.pdf report.txt
# 3) grep 逐个核对关键数字（照抄原文，记页码）
grep -nE "453\.21|6,476,855,244\.59|88\.66|127,673" report.txt
```
可以派研究子 agent 广搜 + 初筛，但**关键数字最后一定亲自 grep 原件核一遍**（子 agent 会转述带偏）。

### 来源分级标注（贯穿备料 → 脚本 → 画面）
- **【官方】** 公司定期报告 / 公告原文
- **【第三方】** 权威媒体 / 研报的分析、时间线
- **【存疑】** 未核实、口径不一、匿名推测

### 红线（越界这期就废）
- 出口管制 / 国产替代 / 宏观利好 = **市场分析**，公司年报把「实体清单」写成的是**供给侧风险**、不是需求利好；不能把市场归因说成公司口径。画面上用**醒目横幅**「⚠ 以下为市场分析 · 非公司口径」隔离整段。
- **匿名客户不猜真身**（"字节/阿里"等媒体推测不作事实）。
- **年报没披露的不编**：云端产品线内部细分收入、具体在售/新型号对收入的贡献、新品量产节点 —— 挖不到就明说「年报只披露到 X 层级」。
- **真人肖像用排版名片**（姓名+头衔+背景卡），不抓网图（版权 + 肖像权，尤其变现内容）；企业/产品图用**官方素材**（官网截图 / 芯片实拍 / 品牌 logo 包），编辑性使用。
- 产出物中**存疑清单单列**，方便复核。

> 备料落盘 `research.md`（分块 + 逐项标源 + 存疑清单），核实脚本落盘，方便下次复用与追溯。

## Step 2 · 脚本（内容标准）

- **提问式开头钩子**（用户偏好）：用一个反差或悬念开场 + 三连问。例：「一家公司，去年还亏 X 亿，今年营收涨了 N 倍、首次盈利 —— 它怎么做到的？靠什么赚钱？谁在买？今天只用公开财报客观拆一拆。本期只做数据分享，不构成投资建议。」**不要**用抽象框架式开头。
- **官方数据为主**，客观陈列；每个数字都能回溯到年报页码 / 报告期。
- **章节化**（归因类）：每章一个小标题，配「章节徽标」（黑底序号 + 黄底标题）；章节起点 = 该段第一句 cue 时间，产出**描述用时间戳**。
- **结尾**：一句话小结（官方口径 + 结构 + 驱动 + 另一面）→ 明说行业叙事是市场分析 → 「以上全部来自公开财报，只做陈列」→ 免责定格（本视频信息均来自公开资料，仅作客观分享，不构成任何投资建议。市场有风险，决策需谨慎）。
- **写作风格**：过 `personal-chinese-writing-style` **两遍**（标点一遍 + 声音一遍），再过**多音字扫描**（喂 TTS 前）。细节委托 blockframe-video 的 Step 0 / listenhub-tts。
- 脚本落盘 `script.md`；同时出**纯口播** `narration.txt`（把百分号写成「百分之 X」、去掉难读的英文型号缩写或改写，利于 TTS）。

## Step 3 · 配音（灵依 1.2x + 逐句精确时间轴 · 重要）

**灵依这个音色的 ListenHub `/v1/speech` 不吐自带字幕**，无法直接拿逐句时间轴。**别用「缩放别的音色字幕来近似」**——会累积漂移、画面滞后（踩过）。正解 **逐句单独合成**：

```
每句单独 POST /v1/speech（{"scripts":[{content,speakerId}]}，端点 api.marswave.ai/openapi/v1/speech）
→ 每句拿回音频，ffprobe 量精确时长 → 按序拼接成整段 → 累加时长 = 每句精确出现时间
→ 据此重排场景与揭示动画 → 整段再 atempo=1.2 提速、SRT 时间除以 1.2
```

驱动脚本见 `scripts/regen_tts.py`（逐句合成 + 拼接 + 出精确 SRT + 1.2x）。零误差、不依赖 ASR、不上屏幕字幕（竖屏短视频不烧字幕，SRT 只用于场景对齐）。出片后**审听**兜多音字。

## Step 4 · 成片（委托 blockframe-video，两版都出）

- **横版 16:9**：`--resolution landscape-4k`（真 4K 超采样）。
- **竖版 3:4**：原生 1080×1440 `--quality high` + lanczos 放大（3:4 无 4K 预设；**别用 `zoom:2`**，见 blockframe-video gotcha）。
- **章节化**：每章一个「章节徽标」场景标记（深挖类）。
- **财经画面组件**（BlockFrame kit 之上）：
  - 数据大字报（营收/净利/毛利 巨字）
  - **正负发散柱**（净利转盈：亏损向下、盈利向上、共享零轴 —— 正负一定要有共享零轴，别把亏损柱和盈利柱各摆各的）
  - 横向**客户榜**条形（前五大逐个金额）
  - 存货/原材料对比柱（备货信号）
  - **风险卡**（全部标「年报原文」）
  - **市场分析横幅**（隔离外部归因段）
  - 官网**芯片/产品实拍图**（官方素材，带「图源」小标）
- 两版共用同一份音频 + 精确时间轴；场景切换按 cue 对齐、零漂移；响度归一 -14 LUFS。

## Step 5 · 封面（4 比例，用户定：3:4 / 4:3 / 16:9 / 9:16）

- **竖版模板**（flex 居中）→ 截 **3:4（1080×1440）** + **9:16（1080×1920）**。
- **横版模板**（左文右图）→ 截 **16:9（1920×1080）** + **4:3（1440×1080）**。
- 内容精简、冲击力强：官方**芯片实拍图** + 巨字公司名 + 一个**杀手级官方数据**（如「营收一年 +453%」）+ **问句钩子**（如「这波爆发，钱从哪来？」）+ 一行免责。**不做图源角标堆砌、不塞满**（用户偏好：信息精简、突出重点）。
- 不用 AI 生成图做数据封面（画不准数字/logo）；codex 图片工具此环境不可用（会谎称成功不落盘）。

## Step 6 · 平台文案

- **视频标题**：悬念 + 客观并存（如「营收一年涨 4 倍多，钱从哪来？｜只看官方财报客观拆解【A股上市公司速览 0X】」）。不用夸张/诱导词。
- **描述**：一段客观概述 + ⏱ 章节时间戳 + 📌 数据来源 + ⚠ 免责声明。

### 公众号文章标题（高点击 × 合规，别用「客观数据盘点」这种抽象无钩子的）

原则：**吸引力和客观合规都要**。抽象定性词（「客观数据盘点」）既无钩子又平；合规区间内最有效的三招 —— **① 具体数字 ② 事实反差 ③ 清单/干货承诺**。「客观 / 只看财报 / 只做陈列」保留为**可信度背书**，不当主钩子。标题 20-30 字、含 ≥1 个数字、可用「」突出、都能加系列前缀「【A股上市公司速览】」。

**一句话红线：标题里只能出现「财报已经发生的事」，绝不能出现「你该拿它怎么办」。**

| ✅ 可用（合规、增强可信） | ❌ 避开（违规 / 毁可信度 / 平台封号） |
|---|---|
| 已披露精确数字（4.5 亿、20 亿、4 倍、28.35%） | 涨跌 / 价格预测（会不会涨、目标价、下一个 X） |
| 会计事实词：扭亏为盈、首次盈利、营收结构、客户集中度、大股东 | 买卖动作暗示：抄底、上车、值得买、机会来了、掘金 |
| 中性事实反差（从亏到盈、原地踏步到翻 4 倍） | 收益承诺 / 夸大：翻倍、暴富、稳赚、财富密码 |
| 来源背书：财报里、年报原文、官网数据、只做陈列 | 震惊体 / 炒作：震惊、惊人、揭秘、内幕、独家爆料 |
| 好奇缺口（首次盈利这一年钱从哪来、99% 来自哪） | 制造焦虑 / 绝对化：必看否则错过、最后机会 |
| 时效稀缺事实：首次、单季、刚披露的一季报 | 诱导性提问（现在能不能买、还能上车吗） |

**填空式标题模板（任何公司套用，`{}` 只填已披露事实与数字，动词永远用中性词 盘/看/摆/理/读懂，绝不填涨跌·买卖·目标价·好坏判断）：**

1. **数字反差**：`【A股上市公司速览】{公司}（{代码}）：从{旧数字}到{新数字}，{一句客观承诺}` — 例：从亏 4.5 亿到赚 20 亿，一家 AI 芯片公司的财报盘点
2. **清单盘点**：`【A股上市公司速览】{公司}（{代码}）：{N}张图/{N}个数字看完{一句行业定位}` — 例：8 张图看完一家国产 AI 芯片公司的财报
3. **好奇缺口**：`【A股上市公司速览】{公司}（{代码}）：{一个反常事实}，{到底/背后是什么}` — 例：65 亿营收里 99% 来自同一条产品线
4. **提问自答**：`【A股上市公司速览】{公司}（{代码}）：{谁/靠什么/卖给谁}？{一句事实答案}` — 例：谁是第一大股东？答案是创始人本人
5. **来源背书**：`【A股上市公司速览】{公司}（{代码}）：只看财报和官网，{中性动词}这家公司`
6. **身份代入**：`【A股上市公司速览】{公司}（{代码}）：关注{行业/主题}的，这份数据值得存一份`
7. **三点事实并列**：`【A股上市公司速览】{公司}（{代码}）：{事实1}、{事实2}、{事实3}` — 例：营收 4 倍、首次盈利、客户高度集中
8. **时效陈述**：`【A股上市公司速览】{公司}（{代码}）：刚披露的{报告名}，{一个精确同比数字}`

出片/发文时**给 3-5 个候选**让用户挑，优先数字反差 + 清单承诺（最稳的合规钩子）。
- **标签**：公司名 / 代码 / A股 / 板块 / 行业 / 财报解读 / 系列名。
- **章节时间戳**：两版通用（同一音频时间轴）。
- 片尾/描述按用户固定模板追加（会员 / 赞助 / 播放列表，见 video-promo 留痕）。

## Gotchas（财经特有）

1. **数字必核原件。** 子 agent 和媒体会把口径带偏（如 2024 归母亏损 4.43 vs 4.52 亿）——以年报对比列原文为准。
2. **市场叙事严格打标签。** H20 管制 → 受益 只能说「市场普遍认为 / 时间线吻合」，公司年报没这么归因。
3. **精确同步用逐句合成**（灵依不吐字幕），别用缩放近似。
4. **正负图要共享零轴**，别让亏损柱和盈利柱各摆各的（会看不出正负）。
5. **色块上的文字用黑色**（BlockFrame）；亏损值标签别用白色（白底上隐形，踩过）。
6. **真人用名片、企业/产品用官方图**（版权 + 肖像权）。
7. **年报没披露的别编**（云端内部细分收入、新型号贡献），明说「只披露到 X 层级」。
