# 平台 · 比例、安全区、裁切

## 比例集（五个，缺一不可）

两种格式**出同一套五个比例**，只有主封面不同。这条是写死的 —— 「又只做了主封面」是这个 skill 存在的头号理由。

| 比例 | 逻辑尺寸 | 主要去处 | 主封面 |
|---|---|---|---|
| `3x4` | 1080×1440 | 抖音 · 小红书 · 视频号 | **short** |
| `9x16` | 1080×1920 | Shorts / 全屏竖屏缩略图 | |
| `16x9` | 1920×1080 | YouTube · 通用缩略图 | **long** |
| `16x10` | 1920×1200 | Bilibili 横版 | |
| `4x3` | 1440×1080 | 横版 4:3 | |
| `1x1` | 1080×1080 | 可选方图 | |

尺寸表的唯一权威是 `scripts/ratios.sh`，渲染和检查共用它。加比例：丢一个 `cover-<ratio>.html`，在 `dims_for()` 加一行。

一律 **2× DPR** 出图（`cover-16x9.png` 落地 3840×2160）。平台会二压，超采样让汉字笔画在压缩后仍然干净。

> **竖版视频本体固定 3:4，不出 9:16。** 9:16 太瘦高、移动端观看体验差（用户明确定下的偏好）。`9x16` 只作为**封面比例**保留。

## 两块吃掉封面的区域

### 右下角：时长徽标（横版）

YouTube 和 Bilibili 在横版缩略图右下角压时长。压在下面的内容直接消失。

保留 **右下 22% × 16%**。`check-covers.sh` 会量这块区域里的墨量，超过 1.5% 就告警（量之前先从画布边缘内缩 4%，避开封面自己的黑描边和硬阴影）。写版的时候给 `#cover` 加 `class="debug"` 就能看见这块红区。

页脚免责声明一律靠**左下**。

### 中心方形：主页宫格（竖版）

抖音 / 视频号 / 小红书的**主页宫格**按中心正方形裁切显示竖版封面。3:4 封面上下各会被切掉约 12.5%。标题必须活在中间那个正方形里。

`check-covers.sh` 把每张封面的 1:1 中心裁切写进 `.cover-check/cover-<ratio>-1x1.png`。**打开看。**

> 各平台的具体像素规格和 UI 覆盖层会变，而且分端（App / Web / 不同版本）不一样。上面两条是**几何上稳的部分**：横版右下有徽标，竖版宫格按方形裁。不要把某个版本的精确 px 抄进代码 —— 留出余量，然后看图。

## 尺寸之外：120px 测试

信息流里的缩略图大约就是 240px 宽。`check-covers.sh` 会把每张封面缩到 240px 写进 `.cover-check/cover-<ratio>-240.png`。

标题在那个尺寸读不出来，这张封面在真实世界就是不工作的 —— 无论它在你的 5K 显示器上多漂亮。治法只有两个：**标题更短**，或者**字更重**。

图示、图表、副标题在 240px 化成纹理，这是**对的**。它们奖励凑近看的人。只有标题必须在缩略图尺寸幸存。
