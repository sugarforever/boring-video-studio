# 构图 · 三个槽位与「主视觉阶梯」

封面不是插画，是**一句话加一个物件**。这份文档是本 skill 的设计内核；模板和脚本只是把它执行出来。

---

## 一、三个槽位

任何一张封面都拆成三格：

```
        ┌──────────────────────────┐
        │                          │   ← 空气（≥ 20%）
        │           ✳              │   MARK        一个物件。可以为空。
        │                          │
        │      Claude Code         │   TITLE       全图最大的墨。点击的理由。
        │                          │
        │   /loop 定时调度任务       │   QUALIFIER   一行上下文。唯一的强调色落在这里。
        │                          │
        └──────────────────────────┘
```

**唯一的规则：任何槽位都可以为空；非空槽位里最大的那个是视觉主体，拿到 display 尺寸，其余一律降级。**

这条规则生成了下面所有版式 —— 新版式是一次槽位配置，不是一个新模板。CSS 里 `--title` 是唯一的自变量，其它字号都由它推导，所以换比例只改一个数。

| 版式 | MARK | TITLE | QUALIFIER |
|---|---|---|---|
| **brand-card** | 官方 logo | 品牌名 | 一句副标题（含强调词）|
| **stat-card** | 巨字数字 | 问句钩子 | 主体 + 出处 |
| **relation-card** | logoA ⊕ logoB | 结果 / 收益 | 组合说明 |
| **glyph-card** | house-style 图形 | 钩子 | 主体 |
| **chip-card** | 标签块（发布 / 实测）| 钩子 | 主体 |
| **title-card** | *（空）* | 钩子 | 主体 |

## 二、主视觉阶梯（没有 logo 时怎么办）

**最容易犯的错，是拿装饰去填 MARK 槽。** 一张 AI 生成的「发光的大脑」不承载信息、破坏系列一致性，而且是图像模型最急于给你的东西。它比留白更糟。

从上往下走，**取第一个成立的档位；一个都不成立，就删掉这个槽**。

### 1 · 官方标
`brand-icons` 有就用。真实的 logo 永远胜过像但不对的。

### 2 · 数字即主视觉
话题的内核是一个量，就把数字提到 MARK 槽、给 display 尺寸，名字降级成注脚。数字不是装饰，它就是点击的理由。

```
巨字 +453%   /   问句「这波爆发，钱从哪来？」   /   公司名降为 qualifier
```

### 3 · 关系即主视觉
话题是两个东西相遇，就把两个标并排、中间放一个运算符。构图本身即论断。注意 TITLE 写**结果**，不是配料表 —— `省 93% 上传成本`，不是 `OpenClaw + Agent Browser`。

### 4 · house-style 图形
有些概念自带典型形态：`/loop` 就是一个斜杠命令，`{}`，一段波形，一个绕回自身的箭头。单线、单色、与 logo 同一笔画粗细。

**只有这一档可以调用 `codex-mark.sh`**，而且只在图标库没有、话题也没暗示出图示的时候。

### 5 · 标签块，无标
发布、实测、深扒 —— 这类没有主语、只有姿态。一个小色块坐在 MARK 槽的位置，标题接管全部重量。

### 6 · 纯字，什么都不放
两级字号、一个刻意的参差、以及空气。**这是一张成立的封面，不是一个需要道歉的兜底。**

## 三、图示可以是主视觉（阶梯之外的上策）

如果这条视频的核心事实**自带形状**，画出来 —— 它比任何图标都强，因为它用第二种模态复述了标题。

真实例子（全志科技 · 无实际控制人）：

```
                    ┌ ─ ─ ─ ─ ─ ┐
                    │    空缺    │      控股股东 / 实际控制人
                    └ ─ ─ ─ ─ ─ ┘
                          │
              ┌───────┬───┴───┬───────┐
              ▢       ▢       ▢       ▢      创始团队 4 人 · 无一控股
```

一个虚线空框、四个等大方块。**纯 CSS。** 观众在读完标题之前就已经看懂了。

这也顺带回答了「图像生成工具在这里做什么」：什么都不做。用 `.mark-card` 包起来。

## 四、让「简单」显得是设计过的

简单和没做完之间，隔着下面这几条。

**视觉居中，不是数学居中。** 堆栈的光学中心落在 46% 高度左右。正正 50% 会读作「偏低」。`.lay-stack` 的非对称 padding 就是干这个的。

**按墨量归一 logo，不是按外框。** 一个星号、一个水滴、一只螃蟹放进同样大的框里，视觉重量差三倍，整个系列的网格就松了。用 `--optical` 逐个微调，眼睛说了算。

**一张封面只有一个强调色。** 落在**让人点击的那个 token** 上：数字、`/loop`、`飞书`。有标就取标自身的颜色，没标就取系列色。BlockFrame 的糖果色块是**结构**（chip、边框、硬阴影），不算强调 —— 但整张图的彩色用法别超过三处。

**强调条用 gradient，不要用 background-color。** `background` 会填满整个行盒；在 shrink-wrap 的 inline-block 上，它会在最后一个汉字处过冲或欠冲。`.hl` 用 `linear-gradient` 精确贴住字形。这是实测踩出来的。

**彩块里的 CJK 要给足 ~0.2em 竖直 padding**，否则字形贴边。

**标题 ≤ 8 个汉字一行，最多两行，参差收边，绝不两端对齐。**

## 五、两个必过的检查

**右下角是平台的。** YouTube、Bilibili 会在横版缩略图右下压一个时长徽标。压在你副标题上的字就是没有了。`check-covers.sh` 会量这块区域的墨量并告警；`#cover.debug` 让你在写的时候就看见它。

**120px 测试。** 把成品缩到 240px 宽（手机信息流里的真实尺寸）再看。标题读不出来，就是标题太长或太轻。图示化成纹理是对的 —— 它奖励凑近看的人，不承担缩略图尺寸的信息。

**竖版还要过 1:1 中心裁切。** 抖音 / 视频号 / 小红书的主页宫格按中心方形裁切显示。标题掉出中间那个正方形，主页就白做了。

`check-covers.sh` 把这两张图写进 `.cover-check/`。**它们是给你看的，不是给脚本看的** —— 没有任何检查能替你判断标题还读不读得出来。

## 六、绝不做的事

- **绝不让图像模型渲染语义文字。** 汉字、数字、logo 一律 HTML。模型会把它们画糊、画错、画成不存在的商标。
- **绝不在生成好的位图上盖字修字。** 不用 ImageMagick / Pillow / SVG 覆盖去补救。改 prompt 重出，或者换一个不含字的版式。
- **绝不用装饰填 MARK 槽。** 见阶梯第 6 档。
- **绝不给一期新视频另起一套设计。** 音色、配色、字体、brandbar 位置、版式都沿用系列。封面是频道的门面，一致性本身就是设计。
