# 多音字清单 · 口播稿过 TTS 前扫一遍

TTS 对**多音字**(一个字多个读音)经常选错读音 - 典型:`调用` 的「调」该读 **diào**(四声),
裸「调」或上下文不清时,引擎常读成 **tiáo**。这份清单 + `scan-heteronyms.sh` 用来在**跑 TTS 前**
把高危字捞出来人工过一遍。

## 怎么用

1. 口播稿定稿后(已过两遍 personal-chinese-writing-style),跑前扫一遍:
   ```bash
   scripts/scan-heteronyms.sh narration.txt
   ```
   它对照下面的清单,把命中高危字的行标 `⚠ REVIEW`(裸字 / 不在已知安全词里)或 `✓`(在安全词里、通常没事)。
2. 对每个 `⚠ REVIEW`,**人工判断**该读哪个音,按这个梯子修:
   - **① 扩成无歧义的词**(首选):`该调什么函数` → `该调用什么函数`;`一行` →(若指代码行)`一行代码`。
   - **② 换种说法**(扩词救不了时):有些字扩成词也会错(「行」很硬),换个词绕开,如 `一行` → `一句` / `单行`。
   - **③ 漏到成片了** → 单句重生成 + 挖补换音轨(见 `SKILL.md` 的「单句修音频」)。
3. 改的是 `narration.txt`,**音频和字幕一起受益**(speech 路字幕=输入原文),不用额外处理。

> **为什么不用引擎的拼音 / SSML 标注?(已实测,2026-06)** ListenHub 原生 `/v1/speech` **不支持**:
> 拼音括号 `调(diào)用` —— 字幕原样保留、音频里 `(diào)` 被静默忽略(不当提示又污染字幕);
> SSML `<phoneme ph="diao4">调</phoneme>` —— 整个标签被当文字读出来(8.85s vs 纯文本 1.13s)。
> 所以**文本改写对这个引擎严格更优**(音字幕同步、不依赖引擎)。换成「字幕独立轨 + 支持 SSML」的引擎(Azure / 阿里 CosyVoice 等)时,才考虑引擎级标注。

> **诚实的限制:** 发音没法从音频自动验证(ASR 转写出的是同一个汉字,分不出 tiáo / diào)。
> 这套是**预防**,不是保证;残余靠审听 + 单句挖补兜底。**不是每个多音字都要改,只在 TTS 会选错时改**,否则稿子会别扭。

## 点号文件名 / 英文缩写的读音(实测 2026-06-26)

带「.扩展名」的名字(如 `DESIGN.md`)直接写进稿子,TTS 把 `.md` 当后缀读成含混的一个音,不是「dot M-D」。同样靠**改写法**解决:

- **解法**:稿子里写成**全小写 + 中文「点」** —— `design 点 md` → 实测读成「design dot M-D」(本项目克隆音色,用户审听确认 v3 正确)。对照过的其它写法都不如它:`DESIGN.md`(含混)、`DESIGN 点 MD`(大写没更好)、`DESIGN 点 markdown`(读成整词 markdown)。
- **字幕回填**:`narration.txt` 用 `design 点 md`;生成 SRT 后做**纯文本替换** `design 点 md` → `DESIGN.md`(只改字、不动时间轴);**屏幕卡片**在合成里直接写 `DESIGN.md`。结果:音对、字幕干净、画面正确。
- **通用做法**:拿不准的点号/缩写名(`a.txt`、`k8s`、`v2`、`gpt-5.5`…),先用本法生成 3–4 个写法变体的小样,**让用户审听**再定(没有 ASR 时这是唯一可靠验证;有本机 whisper-cli 但需自备 ggml 模型)。

## 高危清单(AI / 编程口播领域,遇新坑就加)

| 字 | 正确读音 | 易错成 | 触发 → 怎么写 |
|---|---|---|---|
| 调 | diào | tiáo | 调用 / 调度 = diào;裸「调」「该调…」→ 写「调用」。调试 / 调整 = tiáo,本就对、勿改 |
| 行 | xíng / háng | 互错 | 执行 / 运行 = xíng;一行(代码)= háng。「行」最硬,扩词常救不了 → 换说法(一句 / 单行) |
| 应 | yìng | yīng | 应用 / 响应 / 适应 / 反应 = yìng,常被读 yīng;成词通常能纠 |
| 重 | chóng / zhòng | 互错 | 重试 / 重载 / 重新 / 重复 = chóng;权重 / 重要 = zhòng |
| 长 | cháng / zhǎng | 互错 | 长度 / 长上下文 = cháng;增长 / 成长 = zhǎng |
| 中 | zhòng | zhōng | 命中 / 命中率 = zhòng,常被读 zhōng → 写「命中率」 |
| 处 | chǔ | chù | 处理 = chǔ;别用裸「处」 |
| 量 | liàng | liáng | 向量 / 变量 / 数量 = liàng;测量 = liáng |
| 切 | qiē | qiè | 切换 / 切分 = qiē;一切 = qiè |
| 卡 | kǎ / qiǎ | 互错 | 卡顿 / 显卡 = kǎ;卡住 = qiǎ |
| 给 | gěi / jǐ | 互错 | 交给 / 给出 = gěi;给予 / 供给 = jǐ |
| 差 | chā / chà | 互错 | 差距 / 误差 = chā;差不多 = chà |
| 模 | mó | mú | 模型 / 模式 / 模块 = mó;模样 / 模具 = mú(低危,列着备用) |
| 弹 | tán | dàn | 弹出 / 弹性 = tán;子弹 / 导弹 = dàn |
