# 平台文案 · 结构骨架（YouTube + Bilibili）

`youtube.md` 和 `bilibili.md` 有**大致固定的段落顺序**，两个平台略有不同。本文件给出顺序骨架和占位符 —— 具体内容由 agent 当期填。

> **结构 vs 内容**：这里固定的只有**段落顺序和平台差异**。凡是具体内容 —— 推广、广告、个人信息、相关链接 —— 都是占位符，由 agent 在发布时按当期情况判断：有时有推广、有时没有，保持灵活。内容来源可以是当期对话上下文、用户 memory 里的创作者档案，或直接问用户。

## YouTube · `youtube.md`

```
## 标题（候选）

«2-3 个候选标题»

## 简介

«置顶推广/广告块»       ← 占位符，发布时按情况决定，可有可无
«个人信息块»            ← 占位符，创作者的社群 / 社交 / 联系方式
«正文简介»
«相关链接区»            ← 本期视频分享的相关链接：主题源、上一期视频等

⏱ 章节
0:00 «开场小标题»
m:ss «各章小标题»       ← 见「章节时间戳」
```

## Bilibili · `bilibili.md`

B 站更精简：置顶广告位在最前，不带章节时间戳。

```
## 标题（候选）
（同 YouTube，可完全一致）

## 简介

«置顶推广/广告块»       ← 占位符，发布时按情况决定，可有可无
«个人信息块»            ← 占位符，B 站可只留创作者指定的一两条
«正文简介»
«相关链接区»
```

### 两平台差异一览

| | YouTube | Bilibili |
|---|---|---|
| 章节时间戳 | 有 | 无 |
| 正文 | 同一份 | 同一份，可略压 |
| 推广 / 广告 / 个人信息 | 占位符，发布时判断 | 占位符，发布时判断 |

---

## 标题

标题沿用系列既有口径 —— 观察上几期的实际标题，保持同一调性。候选给 2-3 个。可用的钩子例如：数据 / 星数、系列定位、这期真跑才发现的独家看点。

---

## 章节时间戳

章节结构来自视频的内容规划 —— 口播稿的每个 `[NN]` 小节就是一章（见 SKILL.md「章节化生产」的天然单元），标题按该小节主题写。

每章的起始时间由字幕推理得到：该章对应 `[NN]` 小节的第一句，起始时间 = 那句在 `narration.srt` 里的 cue 起始秒。

- 成片加了封面开篇（cover-as-intro）时，正片整体后推，每章起始时间加上片头偏移。offset = SKILL.md「封面开篇」的转场时刻（对齐开场白结束，约 2.5-3s），以那节为准。
- 首章为 `0:00`（封面那几秒算进开场）；其余每章 = `floor(cue_start + offset)`，格式 `m:ss`。
- 没有封面开篇时 offset = 0，直接用 cue 时间。

SRT 存在就据此构建章节时间戳。改稿重跑 TTS 后，章节结构照旧（还是那几个 `[NN]` 小节），各章起始时间据新 SRT 重新推理即可。

## 正文简介

两平台共用同一份正文 —— 这是唯一约束。具体写法由 agent 自由发挥。

文风沿用创作者的既有写作风格：若用户配了写作风格 skill、或在 memory 里记录了文风偏好，据此来。
