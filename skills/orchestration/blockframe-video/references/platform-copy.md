# 平台文案 · 结构模板（YouTube + Bilibili）

`youtube.md` 和 `bilibili.md` 不是自由写作 —— 它们有**固定的段落顺序**，每个平台还不一样。
以前 Step 5 只说「沿用系列模板」却没写模板在哪，于是每一期都靠临场记忆重建，
必然漂移：正文和推广块前后颠倒、章节忘了算片头偏移、B 站套了 YouTube 的结构……
用户每期都得手动改回来。这个文件把顺序**写死**，消灭那轮返工。

> **通用 vs 私有的分界（重要）**：本文件写死的是**结构、顺序、平台差异、章节偏移规则** ——
> 这些对所有创作者通用。而**具体的推广位内容**（会员链接、赞助商广告词、知识星球 / Twitter
> 链接、播放列表名）是**用户/系列私有**的，不写进本 skill。它们从**用户的推广位配置**读取
> （一份用户维护的 promo 说明，或直接问用户）。本文件里凡是 `«尖括号»` 都是这类占位符。

## YouTube · `youtube.md`

段落顺序**从上到下固定**，推广块在正文**之前**（这是 YouTube 的惯例，也是最容易搞反的一处）：

```
## 标题（候选）

1. «星数钩子» + «系列定位» + «skill 名» - «一句白话收益»
2. …（给 2-3 个，第一条对齐系列既有风格）

## 简介

«会员推广块»            ← 用户私有，整段照搬，放最前

- 加入我的知识星球：«链接»
- 关注我的 Twitter：«链接»
- 如果您有任何问题或者建议，欢迎通过邮件联系我：«邮箱»

«正文简介»              ← 3-4 段：承接上期一句 → 这期讲什么 → 实打实的点 → 本期独家看点

«播放列表名»            ← 例如「【Agent Skills】播放列表」，纯文字，URL 用户在后台挂

- «skill 名» 源：«skill 的 GitHub 目录链接»
- 上一期 «上期 skill 名»：
«上一期视频链接»        ← 系列内每期都回链上一期；没有就留空让用户补

⏱ 章节
0:00 «开场小标题»
m:ss «各段小标题»       ← 见下方「章节时间戳」

## 标签

«逗号分隔，10-12 个：skill 名、作者、平台词、系列词、小木头等»
```

## Bilibili · `bilibili.md`

B 站**更精简**：赞助商广告置顶、**不放会员块、不放章节时间戳、不放个人信息长列表**。

```

（同 YouTube，可完全一致）

## 简介

«赞助商广告»            ← 用户私有，B 站描述第一行，整行照搬

- 加入我的知识星球：«链接»

«正文简介»              ← 和 YouTube 同一份正文，可略压缩

«skill 源链接»

## 标签
«逗号分隔»

«合集名»                ← 例如「Agent Skills」，B 站用合集功能挂
```

### 两平台差异一览

| | YouTube | Bilibili |
|---|---|---|
| 会员推广块 | 有，放最前 | 无 |
| 赞助商广告 | 无 | 有，第一行 |
| 个人信息（星球/Twitter/邮件） | 三条都放 | 只放星球 |

## 标题公式

系列视频的标题**沿用系列既有口径**，别每期另起风格。观察用户上几期的实际标题，套同一个模：

```
«钩子» «系列定位» «skill 名» - «一句白话收益»
```

- **钩子**：星数是最好用的（「Matt的Skills 18万星」）。星数按发布当天核到的真实值，取整到「万」。
- **系列定位**：如「工程链三件套之三」，和上期封面 / 标题的编号连上。
- **白话收益**：一句话说清这期能带走什么，避免抽象（「把 spec 拆成 agent 能一张张开工的工单」优于「深入解析 to-tickets」）。
- 备选标题里可以放一条**独家看点**（这期真跑才发现的东西），往往比通用钩子更抓人。

## 章节时间戳 —— 必须算片头偏移

章节时间戳从 `narration.srt` 的 cue 时间生成：每章起点 = 该章第一句对应的 cue 起始秒。

**关键、最容易漏的一步：若成片加了封面开篇（cover-as-intro），正片被整体后推，所有章节时间戳必须加上片头偏移。**

- 片头做法见 SKILL.md「封面开篇」：封面停留约 3s，`xfade` 从 `offset` 处（默认 2.5s）转入正片。
- 正片旁白的 `t` 秒 → 成片的 `t + offset` 秒。**offset 默认 2.5s。**
- 首章恒为 `0:00`（封面那几秒算进开场）；其余每章 `= floor(cue_start + offset)`，格式 `m:ss`。
- 不加封面开篇则 offset = 0，直接用 cue 时间。

不加这个偏移，整条章节表会系统性早 2-3 秒 —— 观众点章节全部落在上一句话里。

> SRT 的 cue → 章节，和 SKILL.md「SRT 驱动定时」是同一份 SRT 的两个消费者：一个喂动效 `tl`，
> 一个喂章节表。口播稿一改、重跑 TTS，两处都从新 SRT 自动重算，不手工挪时间戳。

## 正文简介写法（两平台共用一份）

四段式，承上启下：

1. **承接上期一句**：系列视频假设观众可能没看上期，用一句话把上期结论带出来（「上期我们用 X 把 Y 做了……」）。
2. **这期讲什么**：一句话定位这个 skill 做的事。
3. **实打实的点**：列 3-4 个这期聊到的具体概念，用短语堆叠，别写成段落。
4. **本期独家看点**：这期真跑 / 真做才碰到的东西（例：真跑一遍才发现 spec 里那条规则会空转）。这一段最值钱，别省。

文风沿用 `personal-chinese-writing-style`：全角标点、单破折号 ` - `、不用网络热词、不用「讲清楚」这类词。
